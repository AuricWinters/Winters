# 编程学习功能整合至实验室模块 - 产品需求文档

## 概述
- **摘要**：将现有的编程学习功能重构并整合至实验室模块，创建一个统一的编程学习环境，包含语言选择、学习任务体系和现代化用户界面。
- **目的**：为用户提供一个集成在实验室环境中的编程学习平台，使其成为实验室生态系统的重要组成部分。
- **目标用户**：初学者和希望复习编程基础知识的用户。

## 目标
- 将编程学习功能完整整合至实验室模块中，确保功能逻辑与实验室环境深度融合
- 重新设计用户界面，使其与Web端视觉风格保持完全一致
- 实现语言选择功能，将语言挑选列表固定展示在侧边栏区域
- 为每个教学单元设计详细的学习任务体系，内容难度控制在基础入门级别

## 非目标（范围外）
- 不改变现有的实验室其他功能模块
- 不增加新的编程语言支持（保持现有的C、Python、JavaScript、PHP、TypeScript、Java、ArkTS）
- 不实现在线编程环境或代码执行功能

## 背景与上下文
- 现有的编程学习功能位于独立的`CodeLearning.vue`组件中
- 实验室模块采用现代化的卡片式布局，具有统一的设计风格
- 项目使用Vue 3 + Vite + Pinia + Vue Router技术栈
- 设计系统基于CSS变量，确保视觉风格的一致性

## 功能需求
- **FR-1**：将编程学习功能整合为实验室模块的一部分
- **FR-2**：实现侧边栏语言选择功能，固定展示语言列表
- **FR-3**：为每个编程语言设计详细的学习任务体系
- **FR-4**：重新设计用户界面，与Web端视觉风格保持一致
- **FR-5**：确保响应式设计，适配不同屏幕尺寸

## 非功能需求
- **NFR-1**：界面视觉风格与实验室模块保持一致
- **NFR-2**：交互逻辑流畅，用户体验良好
- **NFR-3**：代码结构清晰，易于维护和扩展
- **NFR-4**：性能优化，确保页面加载速度快

## 约束
- **技术**：基于现有的Vue 3 + Vite + Pinia + Vue Router技术栈
- **设计**：必须遵循现有的设计系统和CSS变量
- **内容**：保持现有的编程语言支持，不增加新语言

## 假设
- 现有的编程学习内容（教程、示例代码）将被保留和重用
- 用户希望在实验室环境中方便地访问编程学习功能
- 侧边栏语言选择功能将提高用户体验和导航效率

## 验收标准

### AC-1：功能整合
- **Given**：用户访问实验室模块
- **When**：用户点击编程学习卡片
- **Then**：用户进入编程学习界面，与实验室环境无缝集成
- **Verification**：`human-judgment`

### AC-2：语言选择功能
- **Given**：用户进入编程学习界面
- **When**：用户查看侧边栏
- **Then**：用户看到固定的语言选择列表，可随时切换编程语言
- **Verification**：`human-judgment`

### AC-3：学习任务体系
- **Given**：用户选择一种编程语言
- **When**：用户浏览学习内容
- **Then**：用户看到详细的学习任务体系，包含明确的学习目标和操作指引
- **Verification**：`human-judgment`

### AC-4：界面设计
- **Given**：用户访问编程学习界面
- **When**：用户查看界面元素
- **Then**：界面视觉风格与Web端保持完全一致，包括颜色、字体、布局等
- **Verification**：`human-judgment`

### AC-5：响应式设计
- **Given**：用户在不同设备上访问编程学习界面
- **When**：用户调整屏幕尺寸
- **Then**：界面布局自动调整，在移动设备上显示侧边栏为可折叠菜单
- **Verification**：`human-judgment`

- [ ] 侧边栏语言选择在移动设备上的具体实现方式
- [ ] 学习任务体系的详细结构和内容组织方式
