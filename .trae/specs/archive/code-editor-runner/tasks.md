# 教程弹窗添加在线代码编辑器和运行器 - 任务列表

## [x] Task 1: 实现左右分栏布局
- **优先级**: P0
- **依赖**: 无
- **描述**:
  - 修改 `.modal-content` 内部结构，从单栏改为左右分栏
  - 左侧：`.tutorial-panel` (教程内容区，约55%宽度)
  - 右侧：`.editor-panel` (代码编辑器区，约45%宽度)
  - 使用 flexbox 或 grid 布局实现
  - 添加中间分隔线（可拖拽调整宽度，可选）
  - 移动端响应式：改为上下堆叠（编辑器在下）
- **验收标准**: 弹窗正确显示左右两个区域，布局合理
- **测试要求**:
  - `human-judgment`: 分栏布局清晰美观，比例合适

## [x] Task 2: 集成代码编辑器组件
- **优先级**: P0
- **依赖**: Task 1
- **描述**:
  - 选择并集成代码编辑器方案：
    - 方案A（推荐）: Monaco Editor (VS Code同款编辑器)
    - 方案B: CodeMirror (轻量级)
    - 方案C: Ace Editor (成熟稳定)
  - 安装对应npm包
  - 创建 `CodeEditor.vue` 组件封装编辑器
  - 配置语法高亮、行号、主题等
  - 支持语言自动切换（根据当前教程语言）
- **验收标准**: 编辑器正常显示，支持代码输入和基本功能
- **测试要求**:
  - `programmatic`: 编辑器可正常加载和输入
  - `human-judgment`: 编辑器外观专业，操作流畅

## [x] Task 3: 实现代码运行功能
- **优先级**: P0
- **依赖**: Task 2
- **描述**:
  - 设计代码执行方案：
    - 方案A（推荐）: 调用后端API执行代码（需要后端支持）
    - 方案B: 使用 Piston API / Judge0 API 等在线编译服务
    - 方案C: 前端WebAssembly方案（Python用Pyodide, JS直接eval）
  - 创建运行结果展示区域（stdout/stderr）
  - 实现"运行"按钮的点击逻辑
  - 处理超时、错误等异常情况
  - 显示执行状态（运行中/成功/失败）
- **验收标准**: 点击"运行"能执行代码并显示输出结果
- **测试要求**:
  - `programmatic`: Python/JavaScript代码能成功运行并显示输出
  - `human-judgment`: 运行结果清晰易读
