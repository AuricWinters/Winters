# 粒子系统爆炸效果 - 产品需求文档

## Overview
- **Summary**: 为 Winters 网站的粒子系统添加行星爆炸效果，增强视觉体验，并修复边界反弹效果的问题。
- **Purpose**: 解决粒子系统边界反弹效果不明显的问题，同时添加更强烈的爆炸效果，提升网站的视觉吸引力。
- **Target Users**: Winters 网站的访问者，特别是登录页面的用户。

## Goals
- 修复粒子系统的边界反弹效果，确保粒子在碰到页面边界时能够正确反弹
- 添加行星爆炸效果，当用户点击页面时，粒子从点击位置向外飞散
- 增强爆炸效果，使其更剧烈、速度更快、范围更大
- 确保爆炸效果在不同屏幕尺寸下都能正常工作

## Non-Goals (Out of Scope)
- 不修改粒子系统的其他核心功能
- 不添加新的粒子类型或视觉效果
- 不改变现有的鼠标吸引效果

## Background & Context
- 当前粒子系统使用 `src/utils/particles.js` 实现
- 粒子系统已实现基本的边界反弹逻辑，但效果不明显
- 已添加了爆炸效果的基本框架，但需要增强效果
- 登录页面使用 `position: fixed` 的 canvas 元素，需要确保尺寸计算正确

## Functional Requirements
- **FR-1**: 修复粒子系统的边界反弹效果，确保粒子在碰到页面边界时能够正确反弹
- **FR-2**: 实现行星爆炸效果，当用户点击页面时，粒子从点击位置向外飞散
- **FR-3**: 增强爆炸效果，使其更剧烈、速度更快、范围更大
- **FR-4**: 确保爆炸效果在不同屏幕尺寸下都能正常工作

## Non-Functional Requirements
- **NFR-1**: 爆炸效果应当流畅，不影响页面性能
- **NFR-2**: 爆炸效果应当视觉上震撼，符合用户期望
- **NFR-3**: 边界反弹效果应当自然，符合物理规律

## Constraints
- **Technical**: 使用现有的粒子系统代码，不引入新的依赖
- **Business**: 保持与现有网站风格的一致性
- **Dependencies**: 依赖于现有的 Canvas API 和 JavaScript 动画系统

## Assumptions
- 用户使用现代浏览器，支持 Canvas API
- 网站运行在正常的网络环境中
- 粒子系统的性能足够支持爆炸效果

## Acceptance Criteria

### AC-1: 边界反弹效果修复
- **Given**: 粒子系统已初始化，粒子正在运动
- **When**: 粒子碰到页面边界
- **Then**: 粒子应当正确反弹，改变运动方向
- **Verification**: `human-judgment`
- **Notes**: 反弹效果应当自然，符合物理规律

### AC-2: 爆炸效果实现
- **Given**: 粒子系统已初始化
- **When**: 用户点击页面
- **Then**: 粒子应当从点击位置向外飞散，模拟爆炸效果
- **Verification**: `human-judgment`
- **Notes**: 爆炸效果应当明显，粒子应当向各个方向飞散

### AC-3: 爆炸效果增强
- **Given**: 粒子系统已初始化
- **When**: 用户点击页面
- **Then**: 粒子应当以高速向外飞散，覆盖较大范围，具有明显的爆炸效果
- **Verification**: `human-judgment`
- **Notes**: 爆炸效果应当剧烈，速度快，范围大

### AC-4: 不同屏幕尺寸兼容性
- **Given**: 粒子系统已初始化，页面在不同屏幕尺寸下
- **When**: 用户点击页面
- **Then**: 爆炸效果应当在不同屏幕尺寸下都能正常工作
- **Verification**: `human-judgment`
- **Notes**: 爆炸效果应当适应不同的屏幕尺寸
