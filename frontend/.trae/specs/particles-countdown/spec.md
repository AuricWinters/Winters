# 粒子脱离倒计时功能 - 产品需求文档

## Overview
- **Summary**: 为 Winters 网站的粒子系统添加脱离倒计时功能，确保粒子只有在到达光标最中心点时才开始倒计时，优化粒子的吸引和脱离行为。
- **Purpose**: 解决粒子系统中脱离倒计时时机不准确的问题，使粒子在真正到达光标中心后才开始倒计时，提升用户体验的流畅度和自然感。
- **Target Users**: Winters 网站的访问者，特别是登录页面的用户。

## Goals
- 实现粒子在到达光标最中心点时才开始脱离倒计时
- 确保粒子的吸引和脱离行为更加自然流畅
- 优化粒子系统的用户体验
- 与现有的粒子爆炸效果和边界反弹效果集成

## Non-Goals (Out of Scope)
- 不修改粒子系统的核心物理计算逻辑
- 不添加新的粒子视觉效果
- 不改变现有的鼠标吸引半径和强度参数

## Background & Context
- 当前粒子系统使用 `src/utils/particles.js` 实现
- 粒子系统已实现鼠标吸引效果和爆炸效果
- 目前的脱离倒计时逻辑可能在粒子未完全到达光标中心时就开始计时
- 登录页面使用 `position: fixed` 的 canvas 元素

## Functional Requirements
- **FR-1**: 实现粒子到达光标最中心点的检测逻辑
- **FR-2**: 确保粒子只有在到达光标最中心点时才开始脱离倒计时
- **FR-3**: 保持粒子吸引和脱离行为的自然流畅
- **FR-4**: 与现有的粒子爆炸效果和边界反弹效果集成

## Non-Functional Requirements
- **NFR-1**: 脱离倒计时功能应当流畅，不影响页面性能
- **NFR-2**: 粒子的吸引和脱离行为应当自然，符合用户期望
- **NFR-3**: 功能应当在不同屏幕尺寸下都能正常工作

## Constraints
- **Technical**: 使用现有的粒子系统代码，不引入新的依赖
- **Business**: 保持与现有网站风格的一致性
- **Dependencies**: 依赖于现有的 Canvas API 和 JavaScript 动画系统

## Assumptions
- 用户使用现代浏览器，支持 Canvas API
- 网站运行在正常的网络环境中
- 粒子系统的性能足够支持新功能

## Acceptance Criteria

### AC-1: 粒子到达光标中心检测
- **Given**: 粒子系统已初始化，粒子正在被鼠标吸引
- **When**: 粒子到达光标最中心点
- **Then**: 粒子应当开始脱离倒计时
- **Verification**: `human-judgment`
- **Notes**: 粒子应当在真正到达光标中心后才开始倒计时

### AC-2: 脱离倒计时功能
- **Given**: 粒子已到达光标最中心点
- **When**: 倒计时结束
- **Then**: 粒子应当逐渐脱离光标，恢复原始运动
- **Verification**: `human-judgment`
- **Notes**: 脱离过程应当自然流畅

### AC-3: 集成与兼容性
- **Given**: 粒子系统已初始化，页面在不同屏幕尺寸下
- **When**: 用户移动鼠标或点击页面
- **Then**: 粒子的吸引、倒计时和脱离行为应当在不同屏幕尺寸下都能正常工作
- **Verification**: `human-judgment`
- **Notes**: 功能应当适应不同的屏幕尺寸

- [ ] 脱离倒计时的具体持续时间需要根据实际效果调整
